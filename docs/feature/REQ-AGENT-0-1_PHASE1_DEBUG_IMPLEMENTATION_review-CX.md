# REQ-AGENT-0-1 Phase 1 디버깅 로깅 검토 (CX)

## 주요 피드백
1. **DEBUG 로그 노출 조건이 문서화되지 않음**  
   - 문서가 추가한 로깅 대부분이 `logger.debug` 레벨인데, 기본 로거 레벨이 WARNING/INFO이면 실제 사내 실행에서 로그가 수집되지 않습니다.  
   - *개선*: 사내 수집 절차에 `LOG_LEVEL=DEBUG`(또는 `PYTHON_LOGLEVEL=DEBUG`) 설정과 로그 핸들러(파일/콘솔) 예시를 명시해 주세요. 그래야 Phase 1 로그가 실제로 남습니다.

2. **요청 상관키가 로그에 없어 다중 요청 시 추적 곤란**  
   - 현재 로그는 모델명/step 정도만 남기고 `session_id`, `survey_id`, `round_id` 등 식별자를 찍지 않아 동시 실행 시 어떤 로그가 어떤 요청인지 분리하기 어렵습니다.  
   - *개선*: 각 Phase-1 디버그 라인에 `session_id`와 `round_id`(가능하면 `survey_id`)를 포함하거나, 최소한 진입 시 하나의 prefix로 찍어 주세요. PII가 우려되면 해시/마스킹을 사용해도 됩니다.

3. **수집 절차에 로그 저장 위치/보존 방식 안내 부재**  
   - CLI 예시만 있고, 어디에 파일로 저장할지/회차별로 구분할지(예: `logs/deepseek_phase1_YYYYMMDD_HHMM.log`)가 명시되지 않아 나중에 공유·비교가 번거롭습니다.  
   - *개선*: 로그 디렉터리/파일명 패턴을 제안하고, 필요 시 `LOG_FORMAT`(타임스탬프 포함) 예시를 추가해 주세요. INFO/DEBUG 혼합 시 가독성 확보를 위해 구분선이나 prefix 규칙도 간단히 적어두면 좋겠습니다.
